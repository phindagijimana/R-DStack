# Beginner

> You **learn concepts.**

The beginner stage is where you build the vocabulary, the basic tooling, and the first experience of doing R&D end-to-end on something tiny.

## Focus

- What is the field?
- What problem exists?
- What tools are used?
- What data exists?
- What basic methods work?

These are five questions that each take a few weeks to answer in any depth. The goal is not mastery; it is orientation.

## What you actually do

- Read the field's main review article — at least once, slowly.
- Take an online course or work through a tutorial in the methods that matter most.
- Set up a working environment: Python, git, a code editor, a place to write notes.
- Touch real data — even tiny data — using the field's standard formats.
- Write a single end-to-end script that produces a number or a plot.
- Show that script to one person who has done R&D before, and listen to their feedback.

## Example output

> *A simple script that analyses MRI volumes.*

Concretely: a Python script that loads a NIfTI image, runs a known segmentation tool, computes a regional volume, and writes a CSV. Twenty lines of code. One subject. One README.

That is enough. The point is that you held a small piece of R&D in your hands.

## Signal that you are ready for the next stage

You have crossed into Junior R&D when:

- You can read a paper's methods section and roughly imagine the code that would implement it.
- You can install a published tool on your own machine and run it on example data without help.
- You can describe, in your own words, the difference between research and development.
- You have at least one finished tiny project on GitHub that someone else could clone and run.

If those four are true, move on to [Junior R&D](junior.md). If not, do not rush — beginner instincts are the foundation everything else sits on.

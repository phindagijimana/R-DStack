**R&DStack** is an open reference for understanding what Research & Development actually is, how it is done, and how someone grows from a beginner asking "what is R&D?" all the way to a PhD-level contributor who can define and lead a research direction.

It is written for graduate students, research engineers, clinicians, data scientists, and developers entering biomedical AI — anyone who needs a single place that explains:

- What **research** is (asking "what is true?") and what **development** is (asking "what can we build?").
- The eight-step **R&D process** used in real labs and teams.
- What changes when you move from beginner work to **PhD-level** contribution.
- How the **R&D mindset** combines scientific, engineering, validation, and translation layers.
- How all of this looks in **biomedical AI** specifically — with worked examples like hippocampal-sclerosis biomarker discovery from MRI.

## Read it locally

This site is built with [MkDocs Material](https://squidfunk.github.io/mkdocs-material/).

```bash
pip install mkdocs-material
mkdocs serve     # preview at http://127.0.0.1:8000
mkdocs build     # build the static site into site/
```

## What's inside

- `docs/index.md` — landing page and navigation overview.
- `docs/overview/` — what R&D is, and how research, development, and R&D differ.
- `docs/beginner/` — what R&D means at the beginner level.
- `docs/intermediate/` — the eight-step R&D process, one page per step.
- `docs/advanced/` — R&D as innovation: deeper research, stronger development.
- `docs/phd/` — originality, rigor, depth, contribution, independence.
- `docs/growth-path/` — beginner → junior → intermediate → advanced → PhD.
- `docs/mindset/` — scientific, engineering, validation, translation layers.
- `docs/biomedical-ai/` — what R&D looks like in biomedical AI.
- `docs/case-studies/` — concrete worked examples.
- `docs/glossary.md` and `docs/further-reading.md`.
